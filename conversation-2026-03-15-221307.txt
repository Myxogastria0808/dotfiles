  ---
  コードを全部確認した結果、怪しい変更は2つあります：

  1. networking.nftables.enable = true の場所 (incus.nix)

  旧設定では configuration.nix に直接書いていたものが incus.nix
  に移動しました。内容は同じですが、nftables + Docker 
  の組み合わせは既知の問題があります。Docker はデフォルトで iptables
  を使うため、nftables が有効だと起動時に Docker daemon
  が失敗してサービスが詰まることがあります。

  2. services.scx (CPU スケジューラー)

  変更は入ってないですが、念のため—scx_bpfland
  が新しいカーネルと相性が悪くなった可能性もゼロではありません。

  ---
  ログを見て incus.service や docker.service あるいは wg-quick@wg0.service が
  hang していれば、次の対処で確認できます。

  nixos/configuration.nix の一時診断コマンド（壊れた世代でブートする前に）：

  # ブートローダーで壊れた世代を選択 → 起動中に黒画面になったら
  # Ctrl+Alt+F2 で TTY に切り替えて:
  sudo systemctl status incus.service
  sudo systemctl status docker.service
  sudo systemctl status display-manager.service
  sudo journalctl -xb | tail -100
