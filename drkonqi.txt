● failed unitがゼロになりました。対処の内容：

  drkonqiとは
  KDE Plasmaのクラッシュレポーター。KDEアプリがクラッシュした際にダイアログ表示
  ・バックトレース収集を行うKDE専用コンポーネントです。

  なぜグローバルマスクが問題か
  ~/.config/systemd/user/ に mask を置くと全セッションに影響し、KDE Plasma
  でのクラッシュ通知が消えます。

  採用した対策
  drkonqi-coredump-launcher.socket.d/kde-only.conf という drop-in
  を作成し、ConditionEnvironment=XDG_CURRENT_DESKTOP=KDE を追加しました：

  - KDE Plasmaセッション: XDG_CURRENT_DESKTOP=KDE → 条件一致 →
  ソケット有効、drkonqi 正常動作 ✓
  - Hyprlandセッション: XDG_CURRENT_DESKTOP=Hyprland → 条件不一致 →
  ソケット非アクティブ → カスケード発生しない ✓

❯ ちなみに、なんでCrashしてたの？                                               

● クラッシュの連鎖を図解します。

  ---
  ① 発端: hm 実行時に xdg-desktop-portal-hyprland が再起動される

  hm の reloadSystemd ステップで systemctl --user daemon-reload
  が走り、サービスのユニットファイルのパスが変わっていると systemd
  がサービスを再起動します（Restart=on-failure が設定されている）。

  ---
  ② xdg-desktop-portal-hyprland 1.3.11 のデストラクタバグ

  再起動のためサービスが停止するとき、デストラクタが走ります。

  exit()
    → CPortalManager::~CPortalManager()
      → CCWlOutput::~CCWlOutput()
        → wl_proxy_marshal_flags()   ← wl_output.release を送ろうとする
          → wl_map_insert_at()       ← SIGSEGV 💥

  Wayland接続がすでに切断されているのに、デストラクタがプロキシを解放しようとし
  てSEGFAULTします。これはアップストリームのバグで、v1.3.11時点では未修正。

  ---
  ③ drkonqi 6.6.2 自身もバグ持ち

  ②のクラッシュで coredump が生成され、drkonqi-coredump-launcher.socket
  がそれを受け取り、drkonqi が起動します。しかし drkonqi 6.6.2 も
  Hyprland（非KDE）環境で通知を送ろうとすると QTextHtmlParser 内でSEGFAULT。

  ---
  ④ フィードバックループ

  xdg-desktop-portal クラッシュ
    → coredump 生成
      → drkonqi 起動
        → drkonqi もクラッシュ
          → coredump 生成
            → 次の drkonqi 起動
              → またクラッシュ
                → ...（20+ 回）

  drkonqi のソケットに TriggerLimitBurst=0（制限なし）と MaxConnections=16
  が設定されており、上限に達するまで延々と繰り返されていました。

  ---
  根本原因のまとめ

  ┌───────────────────────┬────────────────────────────┬───────────────────┐
  │          層           │            原因            │       状態        │
  ├───────────────────────┼────────────────────────────┼───────────────────┤
  │ xdg-desktop-portal-hy │ デストラクタでWayland解放  │ アップストリーム  │
  │ prland 1.3.11         │ バグ                       │ 未修正            │
  ├───────────────────────┼────────────────────────────┼───────────────────┤
  │ drkonqi 6.6.2         │ 非KDE環境で通知送信時にク  │ アップストリーム  │
  │                       │ ラッシュ                   │ のバグ            │
  ├───────────────────────┼────────────────────────────┼───────────────────┤
  │ ソケット設定          │ TriggerLimitBurst=0        │ 仕様（意図的）    │
  │                       │ で無制限起動               │                   │
  └───────────────────────┴────────────────────────────┴───────────────────┘

  今回の修正は「drkonqiのソケットをHyprlandセッションでは起動させない」というも
  のなので、①②のバグ自体は残っています。ただ、xdg-desktop-portal-hyprland が停止
  時にクラッシュするのは終了処理中なのでサービスの動作には実害はなく、upstream
  でいつかパッチが当たるのを待つ状態です。
